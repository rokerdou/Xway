# 错误信息显示修复

## 问题描述

用户反馈：日志中已经显示了详细的错误信息：
```
ERROR client_core::proxy: 步骤1失败: 端口127.0.0.1:1081已被占用，请检查是否有其他程序正在使用
```

但是**GUI界面只显示了"错误"，没有显示具体的错误信息**。

## 根本原因

前端代码中：
- ✅ 定义了 `error` state（第12行）
- ✅ 使用了 `setError(error)` 设置错误（第89、95行）
- ❌ **完全没有在UI中渲染error信息**

## 修复方案

### 在状态指示器后添加错误信息显示组件

```jsx
{/* 错误信息显示 */}
{error && (
  <div className="bg-red-900/30 border border-red-700 rounded-lg p-3">
    <div className="flex items-start gap-2">
      <span className="text-red-400 text-lg">⚠️</span>
      <div className="flex-1">
        <p className="text-xs font-semibold text-red-400 mb-1">错误提示</p>
        <p className="text-xs text-gray-300 break-all">{error}</p>
      </div>
      <button
        onClick={() => setError(null)}
        className="text-red-400 hover:text-red-300 text-xs"
      >
        ✕
      </button>
    </div>
  </div>
)}
```

### UI效果

修复前：
```
+-------------------------------+
| ● SOCKS5 代理                 |
| ● 启动失败                    |
+-------------------------------+
```

修复后：
```
+-------------------------------+
| ● SOCKS5 代理                 |
| ● 启动失败                    |
+-------------------------------+
| ⚠️ 错误提示                   |
| 端口127.0.0.1:1081已被占用，  |
| 请检查是否有其他程序正在使用  |
|                       ✕      |
+-------------------------------+
```

## 功能特性

1. **自动显示**：当 `error` state不为null时自动显示
2. **详细信息**：显示完整的错误信息
3. **可关闭**：点击 ✕ 按钮可以关闭错误提示
4. **醒目样式**：红色边框和背景，清楚提示错误
5. **文本换行**：`break-all` 确保长错误信息正确换行
