# 正确的双GUI测试方法

## ⚠️ 重要：常见的测试错误

### 错误做法1：启动GUI但没有点击"启动"按钮

```bash
# ❌ 错误
cargo run --bin client-gui-tauri &
cargo run --bin client-gui-tauri &
# 结果：两个GUI都不会监听端口，因为都没有点击启动
```

**问题**：GUI启动后，端口1081不会被自动监听，需要用户点击"启动"按钮。

### 错误做法2：查看日志不完整

```bash
# ❌ 错误
tail /tmp/tauri.log  # 可能只看到部分日志
```

**问题**：日志可能被缓冲，没有实时输出。

## ✅ 正确的测试步骤

### 步骤1：启动第一个GUI

```bash
killall client-gui-tauri 2>/dev/null
cargo run --bin client-gui-tauri 2>&1 | tee /tmp/tauri1.log &
```

### 步骤2：在第一个GUI中点击"启动"按钮

**关键**：必须点击GUI界面中的"启动"按钮！

### 步骤3：验证第一个GUI的代理已启动

```bash
# 方法1：检查端口
lsof -i :1081 | grep LISTEN

# 方法2：查看日志
grep "绑定成功\|SOCKS5代理客户端已启动" /tmp/tauri1.log
```

**预期输出**：
```
✓ 端口127.0.0.1:1081被监听
```

或者日志中应该有：
```
INFO 尝试绑定端口: 127.0.0.1:1081
INFO ✓ 端口绑定成功: 127.0.0.1:1081
INFO SOCKS5代理客户端已启动
```

### 步骤4：启动第二个GUI

```bash
cargo run --bin client-gui-tauri 2>&1 | tee /tmp/tauri2.log &
```

### 步骤5：在第二个GUI中点击"启动"按钮

**预期结果**：
- GUI界面显示错误提示：**"端口127.0.0.1:1081已被占用，请检查是否有其他程序正在使用"**
- 状态显示：**"启动失败"**

### 步骤6：验证第二个GUI的日志

```bash
cat /tmp/tauri2.log | grep -E "(尝试|绑定|失败|ERROR)"
```

**预期输出**：
```
INFO 开始启动代理客户端...
INFO 步骤1: 绑定端口...
INFO 尝试绑定端口: 127.0.0.1:1081
ERROR 绑定端口失败: ...
INFO 步骤1失败: 端口127.0.0.1:1081已被占用，请检查是否有其他程序正在使用
```

## 🔍 排查问题

### 如果第二个GUI没有显示错误

#### 检查1：第一个GUI是否真的启动了代理？

```bash
lsof -i :1081
```

如果没有输出，说明第一个GUI的代理没有启动！
**解决**：在第一个GUI中点击"启动"按钮。

#### 检查2：日志是否完整？

```bash
# 查看完整日志
cat /tmp/tauri1.log
cat /tmp/tauri2.log

# 实时查看日志
tail -f /tmp/tauri1.log
```

#### 检查3：是否同一个GUI进程？

```bash
ps aux | grep client-gui-tauri
```

应该看到2个不同的进程。

### 如果日志中只有"尝试绑定端口"

这说明 `TcpListener::bind()` 调用后没有返回，或者日志缓冲了。

**检查**：
1. 等待几秒钟，让程序完成执行
2. 再次查看日志
3. 或者检查程序是否卡住
