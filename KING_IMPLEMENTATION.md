# King加密算法 - 简化实现说明

## 🎯 核心原理

采用**基于位置轮换的映射表替换**方案：

### 加密过程

```
输入: [0x41, 0x42, 0x43] ("ABC")
加密:
  第0字节 → 使用映射表0 → ENMAP[0x41] = 0x6A
  第1字节 → 使用映射表1 → ENMAP[0x42] = 0x2D
  第2字节 → 使用映射表2 → ENMAP[0x43] = 0x10
  第3字节 → 使用映射表3 → ENMAP[0x44] = ...
  ...
  第7字节 → 使用映射表0 → ENMAP[0x4B] = ... (循环)

输出: [0x6A, 0x2D, 0x10, ...]
```

### 解密过程

```
输入: [0x6A, 0x2D, 0x10]
解密:
  第0字节 → 使用映射表0 → DEMAP[0x6A] = 0x41
  第1字节 → 使用映射表1 → DEMAP[0x2D] = 0x42
  第2字节 → 使用映射表2 → DEMAP[0x10] = 0x43

输出: [0x41, 0x42, 0x43] ("ABC")
```

## 📊 算法特点

| 特性 | 说明 |
|------|------|
| **类型** | 替换密码（Substitution Cipher） |
| **映射表** | 7张256字节的映射表（可循环使用） |
| **密钥** | 映射表本身 + 起始索引 |
| **复杂度** | O(n) 时间，O(1) 空间 |
| **安全性** | 中等（取决于映射表的随机性） |

## 🔧 技术实现

### 数据结构

```rust
pub struct KingObj {
    encode_index: usize,  // 加密索引（0-255循环）
    decode_index: usize,  // 解密索引（0-255循环）
    seed: u8,             // 初始种子
    // ... 其他状态字段
}
```

### 核心函数

```rust
// 加密
encrypted[i] = ENMAPS[(encode_index + i) % 7][data[i]]

// 解密
decrypted[i] = DEMAPS[(decode_index + i) % 7][data[i]]
```

## 📈 性能优势

1. **快速**：每个字节只查一次表，无复杂计算
2. **简单**：易于理解和实现
3. **可扩展**：可以轻松扩展到更多映射表
4. **内存友好**：只需存储7张256字节的映射表

## 🔒 安全性说明

### 优点
- 隐藏了原始数据的字节值
- 不同位置使用不同映射表，增加分析难度
- 可以定期更换映射表提高安全性

### 限制
- 单纯替换密码可能被频率分析攻击
- 需要确保映射表的随机性
- 不如现代加密算法（如AES）安全

### 建议用途
- ✅ 数据混淆和轻度加密
- ✅ 防止普通用户窥探
- ✅ 与其他加密方式结合使用
- ❌ 不建议用于高安全性场景

## 📝 使用示例

```rust
use shared::crypto::KingObj;

// 创建加密对象
let mut king = KingObj::new();

// 加密数据
let mut data = b"Hello, World!".to_vec();
king.encode(&mut data, data.len())?;

// 解密数据
let mut king2 = KingObj::new();
king2.encode_index = king.encode_index; // 使用相同的索引
king2.decode(&mut data, data.len())?;

assert_eq!(data, b"Hello, World!");
```

## 🚀 未来改进

1. **扩展为7张独立映射表**：
   - 当前：所有位置使用同一张ENMAP/DEMAP
   - 目标：每张映射表都不同

2. **映射表生成**：
   - 可以基于密钥动态生成映射表
   - 使用伪随机数生成器创建随机置换

3. **性能优化**：
   - 使用SIMD指令并行处理
   - 预取映射表提高缓存命中率

## 总结

这是一个**简单高效的流加密实现**，通过按位置轮换映射表来增加安全性。虽然不如现代加密算法强大，但对于基本的加密需求已经足够。
