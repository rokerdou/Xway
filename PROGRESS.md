# SOCKS5加密隧道系统 - 实现进度

## 架构说明

这是一个**加密隧道**系统，类似于Shadowsocks：

```
本地应用 → 本地SOCKS5客户端(加密) → 远程服务端(解密) → 目标服务器
         ↑                              ↓
         └──────── 加密数据返回 ───────────┘
```

**关键特性**：
- ✅ 流加密：每个字节独立加密/解密
- ✅ 自定义协议：长度前缀 + 加密数据
- ✅ 双向加密：客户端↔服务端全程加密

## 🔄 数据流程

### 客户端（本地）
```
1. 本地应用连接到 127.0.0.1:1081
2. SOCKS5握手
3. 读取目标地址 (example.com:80)
4. 加密目标地址 → 发送到远程服务端
5. 接收本地数据 → 加密 → 发送到远程
6. 接收远程数据 → 解密 → 发送给本地应用
```

### 服务端（远程）
```
1. 接收客户端连接
2. 接收加密的目标地址 → 解密
3. 连接到目标服务器 (example.com:80)
4. 接收客户端数据 → 解密 → 发送到目标
5. 接收目标数据 → 加密 → 发送给客户端
```

## 🔐 加密实现

### 当前实现（简化版）
使用XOR流加密作为占位符：
```rust
let encrypted = data[i] ^ self.seed ^ (idx as u8);
```

## ⚙️ 配置说明

### 客户端配置 (config/client.toml)
```toml
[server]
remote_server = "127.0.0.1"  # 远程服务端地址
remote_port = 1080            # 远程服务端端口

[local]
listen_addr = "127.0.0.1"     # 本地监听地址
listen_port = 1081            # 本地SOCKS5端口
```

### 服务端配置 (config/server.toml)
```toml
[server]
listen_addr = "0.0.0.0"       # 监听所有接口
listen_port = 1080            # 监听端口

[relay]
max_buffer_size = 8192        # 缓冲区大小
```
